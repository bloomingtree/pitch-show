# 开发注意事项
这是一个开发注意事项，在开发过程中一定要严格遵守这些规则，这些规则用来让代码更加健壮，更加符合个人规范。

- 一个页面的代码不宜超过700行。在开发一个弹窗或者侧边栏时，这个部分与主页面的交互比较好分离时，最好将其抽象成为一个组件。
- 不要将svg写入代码中，应当在/components/icons下，针对每一个svg创建一个文件，然后创建一个总的文件来能够调用这些图标。

## git commit 格式
- 使用ADD（添加功能） MOD（修改功能实现） DEL（删除某些代码） FIX（修复bug） UI（优化界面）这些来识别码此次代码提交
- 在识别码后面紧跟具体的内容，中文，简洁明确。如果实在一句话写不完，可以在后面回车紧跟一个md格式列表
- 一次提交涉及到多个操作的，分行编写，每一行都有一个识别码+具体内容
### 示例
示例一：
[MOD]取消帮助栏图片
示例二：
[ADD]增加导航栏 
[FIX]修缮音频分离后的波形展示页面 
[ADD]关于页面增加内容 
[ADD]增加SEO优化
示例三：
[UI] 优化界面交互和配色方案
- 重构播放进度条样式，使用圆角渐变设计和白色指示器
- 优化时间显示格式，采用等宽数字字体提升可读性

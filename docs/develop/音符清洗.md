钢琴卷帘音符显示控制面板 – 需求文档
1. 功能概述
在现有音频分析工具的钢琴卷帘视图上，增加一个控制面板，允许用户实时调节音符的显示方式，以过滤杂乱音符、提升可读性。面板提供三个核心控制项，并支持将“音符融合”算法独立封装，便于后续优化。

如果需要更精细控制，可增加以下两项（默认隐藏，点击“高级”展开）：

合并间隔 (mergeGap)：两个音符之间最大允许的静音间隔（秒），默认 0.05 秒。

音高容差 (pitchTolerance)：合并时允许的最大音高差异（半音数），默认 1（即允许相差一个半音的音符合并）。

2.3 交互行为
任何控件的值改变后，实时更新钢琴卷帘显示（无需点击“应用”按钮）。

为了保证性能，可采用防抖处理：用户停止滑动后 100ms 再执行重绘。

面板底部提供一个 “重置默认” 按钮，一键恢复所有控件到默认值。

可选：提供 “导出当前视图” 按钮，将当前过滤/合并后的音符导出为 MIDI 或 JSON。

1. 音符融合算法要求
3.1 抽离要求
将“音符融合”逻辑单独封装为一个独立的函数/模块，便于后续优化、替换或测试。

函数签名建议：

javascript
function mergeNotes(notes, mergeGap = 0.05, pitchTolerance = 1)
输入：原始音符数组（每个音符包含 start, end, pitch, confidence, amplitude 等属性）。
输出：合并后的新音符数组（不修改原数组）。

算法步骤（参考）：

按 pitch 分组。
每组内按 start 升序排序。
遍历合并：
如果当前音符与前一个音符的 start <= 前一个.end + mergeGap 且 |pitch - 前一个.pitch| <= pitchTolerance，则合并：
新的 start = min(前一个.start, 当前.start)
新的 end = max(前一个.end, 当前.end)
新的 pitch 可简单取平均值（或取出现次数多的）
confidence 可取最大值或平均值
amplitude 可取平均值
否则，前一个音符结束，开始新的分组。
返回合并后的音符列表。
3.2 注意事项
合并时应保留原始音符的其他属性（如 velocity、原始置信度等），但可进行适当合并策略（如取平均）。

合并后的音符可能会跨越原来多个音符，保证时间上连续不重叠。

合并算法应独立于 UI，便于单元测试。

4. 技术实现要点
4.1 前端数据结构
原始音符数据从后端加载后存储为 rawNotes 数组，不可变。

当前显示的过滤/合并结果通过计算属性 displayNotes 动态生成。

每次控件变化时，重新计算 displayNotes，并触发钢琴卷帘重绘。

4.2 性能优化
防抖处理控件变更事件（建议 lodash/debounce 或原生实现）。

如果音符数量极大（>10000），合并算法可能耗时，可考虑将合并放入 Web Worker，但当前阶段可先保证主线程流畅（利用 requestAnimationFrame 分批渲染）。

4.3 与现有代码集成
在钢琴卷帘组件内部或父组件中增加控制面板组件。

控制面板状态通过 Vue/React 状态管理（或原生事件）传递给渲染逻辑。

5. 用户友好性说明
参数命名采用直白的中文描述，避免技术术语（如“置信度”已算常用，但可配合 Tooltip 解释）。

滑块旁边实时显示当前数值，并标明单位（秒、%）。

开关按钮状态用明显颜色区分（如蓝色表示开启）。

提供简短的帮助文本或问号图标，解释每个参数的作用。

示例帮助文本：

最短音符时长：隐藏持续时间太短的音符（通常是杂音）。

最低置信度：隐藏模型不太确定的音符。

自动合并短音符：将相邻或重叠的同音高音符连接起来，减少碎片。

6. 预期效果
用户通过调整面板，可以：

快速隐藏掉原始模型输出的短时杂音。

只显示高置信度的音符，突出主旋律。

开启合并后，钢琴卷帘上原本破碎的音符会变成连贯的长条，更接近真实乐谱。

这一交互方式极大提升了可视化调试效率，也为后续导出乐谱提供了可预览的预处理效果。

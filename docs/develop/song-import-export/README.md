## 1. 功能概述

为 Pitch-Show 新增**歌曲导入导出**功能，实现以下核心能力：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **导出歌曲** | 将分析结果（音符、节拍）导出为文件 | P0 |
| **导入歌曲** | 从文件恢复已分析的歌曲数据 | P0 |
| **批量导出** | 一次性导出多首歌曲 | P1 |
| **批量导入** | 一次性导入多首歌曲 | P1 |
| **格式兼容** | 支持标准 MIDI 格式导出 | P2 |

## 3. 核心决策

### 3.1 导出格式选择

| 格式 | 用途 | 优先级 | 说明 |
|------|------|--------|------|
| **.pitch** (JSON) | 原生格式，完整数据 | P0 | 自定义扩展名，包含所有分析结果 |
| **.mid (MIDI)** | 通用格式，软件互通 | P2 | 标准 MIDI 文件，可在 DAW 中打开 |
| **.musicxml** | 打谱软件格式 | P3 | MuseScore、Sibelius 等 |

### 3.2 技术选型

| 模块 | 方案 | 理由 |
|------|------|------|
| **文件处理** | 原生 File API + Blob | 无需额外依赖 |
| **压缩** | fflate (已有) | 减小文件体积 |
| **MIDI 生成** | midi-writer-js | 轻量级，易于集成 |

### 3.3 设计原则

1. **渐进增强**：先支持原生格式，再扩展通用格式
2. **向后兼容**：导入时自动处理旧版本数据格式
3. **用户友好**：操作简单，错误提示清晰
4. **安全可靠**：数据校验，防止损坏文件导致崩溃

## 4. 快速预览

### 4.1 导出流程

```
用户操作                    系统处理
   │                           │
   ├─ 选择歌曲 ─────────────────→ 获取歌曲数据
   │                           │
   ├─ 点击"导出" ───────────────→ 打包数据
   │                           ├─ 音符数据
   │                           ├─ 节拍数据
   │                           └─ 元数据
   │                           │
   ├─ 选择格式 ─────────────────→ 格式转换
   │                           │
   └─ 保存文件 ←───────────────── 生成下载
```

### 4.2 导入流程

```
用户操作                    系统处理
   │                           │
   ├─ 点击"导入" ───────────────→ 打开文件选择器
   │                           │
   ├─ 选择文件 ─────────────────→ 读取文件
   │                           ├─ 解析格式
   │                           ├─ 校验数据
   │                           └─ 检查冲突
   │                           │
   ├─ 确认导入 ─────────────────→ 写入 IndexedDB
   │                           │
   └─ 查看歌曲 ←───────────────── 刷新列表
```

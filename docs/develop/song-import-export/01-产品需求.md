## 1. 需求背景

### 1.1 现状分析

**当前问题：**

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 数据仅存本地 | 清除浏览器缓存后丢失 | 高 |
| 无法跨设备 | 换设备需要重新分析 | 高 |
| 无法分享 | 无法与他人共享分析结果 | 中 |
| 无法备份 | 没有数据保护机制 | 高 |

**用户反馈（假设）：**

> "我花了好长时间分析完这首歌，结果清除浏览器缓存后全没了..."
> "我想把分析结果发给我的学生，让他参考练习..."
> "我想把音符导入到 Cubase 里进一步编辑..."

### 1.2 市场对标

| 产品 | 导出格式 | 导入功能 | 云同步 |
|------|----------|----------|--------|
| AnthemScore | MIDI, MusicXML | ❌ | ❌ |
| Melodyne | MIDI | ❌ | ❌ |
| Transcribe! | MIDI, XML | ✅ | ❌ |
| **Pitch-Show** | .pitch, MIDI | ✅ | 规划中 |

## 3. 用户场景

### 3.1 场景一：数据备份

**场景描述：** 用户担心数据丢失，希望定期备份分析结果

| 步骤 | 用户行为 | 系统响应 |
|------|----------|----------|
| 1 | 打开歌曲列表 | 显示所有已分析歌曲 |
| 2 | 点击"导出全部" | 弹出格式选择 |
| 3 | 选择"原生格式" | 开始打包 |
| 4 | 确认下载 | 生成 .pitch 文件并下载 |

**验收标准：**
- 支持单首和批量导出
- 导出文件包含完整数据
- 文件大小合理（每首歌约 10-100KB）

### 3.2 场景二：跨设备迁移

**场景描述：** 用户更换设备，需要迁移数据

| 步骤 | 用户行为 | 系统响应 |
|------|----------|----------|
| 1 | 在新设备打开应用 | 显示空列表 |
| 2 | 点击"导入歌曲" | 打开文件选择器 |
| 3 | 选择 .pitch 文件 | 解析并校验 |
| 4 | 确认导入 | 显示导入结果 |
| 5 | 查看列表 | 歌曲显示在列表中 |

**验收标准：**
- 导入后数据完整无损
- 自动处理名称冲突
- 导入失败有明确错误提示

### 3.3 场景三：分享给他人

**场景描述：** 用户想将分析结果分享给朋友

| 步骤 | 用户行为 | 系统响应 |
|------|----------|----------|
| 1 | 选择要分享的歌曲 | 高亮选中 |
| 2 | 点击"导出" | 弹出导出选项 |
| 3 | 选择格式 | 生成对应格式文件 |
| 4 | 通过微信/邮件发送 | （系统外操作） |
| 5 | 对方接收并导入 | 歌曲出现在对方列表 |

**验收标准：**
- 文件可正常传输
- 对方导入后数据一致
- 支持跨平台（Windows/Mac/移动端）

### 3.4 场景四：导出到 DAW

**场景描述：** 音乐制作人想将音符导入到 DAW 进行编曲

| 步骤 | 用户行为 | 系统响应 |
|------|----------|----------|
| 1 | 选择歌曲 | 高亮选中 |
| 2 | 点击"导出为 MIDI" | 格式转换 |
| 3 | 下载文件 | 生成 .mid 文件 |
| 4 | 在 DAW 中导入 | （系统外操作） |

**验收标准：**
- MIDI 文件符合标准
- 音高、时值正确转换
- 支持 DAW 正常识别

## 4. 功能范围

### 4.1 功能清单

#### P0 - 必须实现（MVP）

| 功能 | 描述 | 说明 |
|------|------|------|
| 单曲导出 | 导出单首歌曲为 .pitch 文件 | 核心功能 |
| 单曲导入 | 从 .pitch 文件导入歌曲 | 核心功能 |
| 数据校验 | 导入时校验文件格式和完整性 | 防止错误数据 |
| 冲突处理 | 同名歌曲导入时的命名策略 | 自动添加后缀 |

#### P1 - 应该实现

| 功能 | 描述 | 说明 |
|------|------|------|
| 批量导出 | 一次性导出多首歌曲 | 提升效率 |
| 批量导入 | 一次性导入多首歌曲 | 提升效率 |
| 导出进度 | 大文件导出时显示进度 | 用户体验 |
| 导入进度 | 多文件导入时显示进度 | 用户体验 |

#### P2 - 可以实现

| 功能 | 描述 | 说明 |
|------|------|------|
| MIDI 导出 | 导出为标准 MIDI 格式 | 生态互通 |
| 选择性导出 | 只导出部分数据（如仅音符） | 灵活性 |

#### P3 - 未来考虑

| 功能 | 描述 | 说明 |
|------|------|------|
| MusicXML 导出 | 导出为乐谱格式 | 打谱软件兼容 |
| 云同步 | 自动同步到云端 | 便捷性 |
| 加密导出 | 密码保护导出文件 | 安全性 |

### 4.2 边界定义

**包含：**
- ✅ 音符数据（pitchMidi, startTimeSeconds, durationSeconds 等）
- ✅ 节拍数据（BPM, beats）
- ✅ 元数据（歌名、分析日期、版本）
- ✅ 用户设置（音符过滤器设置）

**不包含：**
- ❌ 原始音频文件（体积过大）
- ❌ 用户账户信息（隐私考虑）
- ❌ 临时状态数据

### 4.3 数据保留策略

| 数据类型 | 是否导出 | 理由 |
|----------|----------|------|
| 音符数据 | ✅ | 核心数据 |
| 节拍数据 | ✅ | 核心数据 |
| 音频波形 | ❌ | 可从原文件重建 |
| 原音频文件名 | ✅ | 参考信息 |
| 分析时间 | ✅ | 元数据 |
| 过滤器设置 | ✅ | 用户偏好 |

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 单曲导出时间 | < 1s | 1000 个音符以内 |
| 单曲导入时间 | < 1s | 含校验 |
| 批量导出（10 首） | < 3s | 含压缩 |
| 批量导入（10 首） | < 5s | 含校验和写入 |
| 文件大小 | < 500KB/首 | 压缩后 |

### 5.2 兼容性要求

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 移动端 | iOS Safari, Android Chrome |
| 数据格式 | 向前兼容 1 个大版本 |

### 5.3 安全性要求

| 要求 | 实现方式 |
|------|----------|
| 数据完整性 | SHA-256 校验和 |
| 格式校验 | Schema 验证 |
| 错误处理 | 优雅降级，不崩溃 |
| 隐私保护 | 不包含用户敏感信息 |
